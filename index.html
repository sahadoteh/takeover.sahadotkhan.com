সাবডোমেইন টেকওভার কি?
সাবডোমেইন টেকওভার (Subdomain Takeover) একটি সাইবার সিকিউরিটি দুর্বলতা, যা তখন ঘটে যখন একটি সাবডোমেইন একটি এক্সটার্নাল সার্ভিসে পয়েন্ট করে, কিন্তু সেই সার্ভিসটি আর সক্রিয় থাকে না বা কনফিগার করা হয়নি। এর ফলে আক্রমণকারী সেই সাবডোমেইনটি দখল করে নিজের কন্টেন্ট বা কোড চালাতে পারে।
কিভাবে সাবডোমেইন টেকওভার হয়?
সাবডোমেইন পয়েন্টিং: একটি সাবডোমেইন (যেমন test.example.com) যদি কোনো থার্ড-পার্টি সার্ভিস (যেমন GitHub Pages, AWS S3, Heroku) ব্যবহার করে সেটআপ করা হয়।
সার্ভিস বন্ধ হওয়া: সেই থার্ড-পার্টি সার্ভিসটি যদি ডিলিট করা হয় বা কনফিগারেশন বন্ধ হয়ে যায়, তবুও সাবডোমেইন DNS এন্ট্রি ঐ সার্ভিসের দিকে পয়েন্ট করে।
আক্রমণ: আক্রমণকারী ঐ সাবডোমেইনের জন্য একই সার্ভিসে নতুন একটি অ্যাকাউন্ট তৈরি করে। এর ফলে সাবডোমেইনটি আক্রমণকারীর নিয়ন্ত্রণে চলে যায়।
